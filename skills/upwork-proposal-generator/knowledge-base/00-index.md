# Knowledge Base Index

This folder is the growth surface for the skill.

Add new experience, proof, positioning, and differentiators here. Do not change the templates unless the user explicitly asks to change the proposal structure.

## Folder map

- `profiles/`: baseline positioning for Vanessa and the agency
- `experience/`: reusable work history and role-specific proof
- `proof-points/`: awards, portfolio, budgets, and other credibility anchors
- `differentiators/`: angles that make the proposal feel distinct
- `job-types/`: mapping for common Upwork job categories
- `rules/`: factual guardrails that must stay true

## How to use the KB

1. Read the profiles first.
2. Load only the experience, proof, and differentiator files that match the brief.
3. Use the job-type file to narrow the best angles.
4. Check `rules/accuracy-guardrails.md` before finalizing.

## How to grow the KB

1. Copy `00-entry-template.md`.
2. Create one new file for one reusable topic.
3. Put it in the right folder.
4. Add `use_when`, safe framing, and avoid-notes.
5. Keep facts verifiable and reusable across future proposals.

## Frozen vs. growing files

Frozen:
- `../assets/templates/*`
- `../references/proposal-rules.md`
- `../SKILL.md` unless the skill behavior itself needs to change

Growing:
- everything inside this `knowledge-base/` folder
