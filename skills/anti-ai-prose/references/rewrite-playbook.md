# Rewrite Playbook

Use this file when the user wants prose that does not sound like ChatGPT, Claude, or Gemini.

## 1. Default rewrite sequence

1. Delete the opener if it only frames, softens, or dramatizes.
2. Mark every phrase from the anti-pattern inventories.
3. Identify the actual claim, decision, objection, or emotional beat.
4. Rebuild the paragraph around that core.
5. Replace abstractions with actors, actions, and consequences.
6. Vary sentence length and stress.
7. Remove the summary sentence unless it adds new force.
8. Recheck against the reference inventories.

## 2. Transformation patterns

### A. Cut the scene-setting opener
Before:
- In today’s fast-paced world, organizations need to rethink how they approach productivity.

After:
- Most teams do not have a productivity problem. They have a prioritization problem.

### B. Replace pseudo-depth with a direct claim
Before:
- At its core, effective leadership is about building trust.

After:
- Leadership fails when people stop believing what the leader says.

### C. Replace mirrored contrast with plain judgment
Before:
- It’s not about working harder, it’s about working smarter.

After:
- More effort does not fix a bad process.

### D. Replace abstract helper verbs
Before:
- This framework enables teams to unlock greater efficiency.

After:
- This process cuts handoff time.

### E. Replace generic uplift endings
Before:
- Ultimately, this approach can unlock new possibilities for long-term success.

After:
- It will save time in one place and create work in another. Decide whether that tradeoff is worth it.

## 3. Drafting from scratch

When generating fresh text, do not produce generic prose first. Start with this sequence:

1. What is the sharpest true claim?
2. What concrete situation makes it real?
3. What objection or tradeoff matters?
4. What language would a smart human actually choose here?

Then draft.

## 4. Tone targets

### Plainspoken analytical
- direct
- unsentimental
- concrete
- selective with transitions

### Commercial but credible
- benefit-led
- concrete
- no hype stack
- no generic empowerment language

### Thought leadership without chatbot residue
- point of view first
- named tradeoffs
- original sequencing
- no theatrical pivots

## 5. Self-check before you answer

Ask:
- Did I keep any stock opener because it sounded smooth?
- Did I replace one cliché with another from the same family?
- Does the passage say one thing three times?
- Are the nouns doing work the verbs should do?
- Does the ending add force or just closure?
- Would a sharp human writer keep this sentence?

## 6. If the user asks for an instruction set or prompt

Return a compact rule system with:
- explicit banned phrase families
- desired stylistic traits
- rewrite workflow
- self-check list

## 7. If the user asks for an exhaustive list

Use all reference files together.
