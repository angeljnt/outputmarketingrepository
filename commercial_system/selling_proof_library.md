# Selling Proof Library

Use this file to pull the right proof for the right package without over-claiming.

## Proof rules

- Translate historical work into capability proof first.
- Do not imply that a historical client literally bought today's packages unless that is true.
- Match proof to the buyer's concern, not just the agency's favorite story.

## No-go proof moves

- Do not use influencer-led work as the default proof for the current public offer.
- Do not use white-label proof as if it proves founder-led SaaS retainers one-to-one.
- Do not use Archive proof as if every capability is equally public and equally proven.
