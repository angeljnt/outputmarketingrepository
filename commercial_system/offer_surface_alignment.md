# Offer Surface Alignment

This file keeps every outward-facing surface aligned to the same offer logic.

## Alignment rules

- The homepage sells the category and the core public offer.
- The pricing page explains packages and add-ons.
- Service pages explain the main capability layers inside the core offer.
- The contact page converts both core SaaS leads and secondary white-label inquiries.
- Outreach can be wider than the homepage, but not contradictory.
- Proposals should follow package logic, proof logic, and ICP logic from this folder.

## Surface map

| Surface | Primary audience | Must emphasize | Can mention | Should avoid |
|---|---|---|---|---|
| `Home` | core B2B SaaS ICP | founder voice + content + SEO + pilot-first | selective background proof, secondary white-label mention in FAQ or lower page sections | full-service positioning, white-label as a headline offer, "no retainer" language |
| `Pricing` | qualified buyers | `30-Day Pilot`, `Foundation`, `Engine`, `Full Stack`, add-ons, month-to-month after pilot | white-label as custom FAQ item | extra public tiers |
| `Service pages` | buyers researching capability depth | one capability family at a time tied to the package system | adjacent support, internal-team support | unrelated service sprawl |
| `Work` | proof-seeking buyers | depth of execution, package-relevant proof, pilot-first logic | why Archive is representative | overclaiming proof breadth |
| `About` | trust-seeking buyers | senior operators, narrow focus, pilot-first model | white-label as secondary custom capability | sounding like a large agency with a broad bench |
| `Contact` | ready buyers | clear entry paths, pilot-first, month-to-month if the fit is right | custom white-label note | vague all-things-to-all-people intake language |
| `Outreach docs` | outbound targeting and copywriting | ICP rules, package fit, pain-first messaging | broader service language for targeting precision | contradicting the public offer architecture |
| `Proposal system` | sales conversations | client benefit, package fit, relevant proof only | custom scopes when the fit is real | credential dumping or package confusion |
| `CRM` | internal operating system | package fit, proof angle, pain point, status | notes on edge cases | free-form fields with no taxonomy |

## White-label alignment rule

White-label is public enough to be found by the right buyer, but quiet enough that it does not reposition the agency in the market.

## Language alignment rule

Use:

- pilot first
- month-to-month after the pilot
- organic pipeline
- founder voice plus company content
- custom white-label support available

Avoid:

- no retainer
- full-service agency
- influencer agency
- generic growth partner language
