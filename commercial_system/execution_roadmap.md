# Execution Roadmap

This roadmap tracks what it takes to leave Output Marketing fully organized around product and marketing.

## Phase 1: Language and naming

### Goal

Make English the only operating language for active product and marketing docs.

### Status

In progress through this cleanup pass.

### Deliverables

- English file names in `commercial_system/`
- English-first source of truth
- retirement of mixed-language operating docs

## Phase 2: Product system

### Goal

Turn the agency into a clear product system instead of a loose services list.

### Status

Implemented structurally.

### Deliverables

- `services_master.md`
- `packages_master.md`
- add-on logic
- public-core vs custom-secondary split

## Phase 3: ICP and segmentation

### Goal

Define who the agency is really for, who is secondary, and who should be excluded.

### Status

Implemented structurally, still needs ongoing enrichment and validation.

### Deliverables

- `client_taxonomy.md`
- `clients_master.csv`
- `prospect_package_mapping.csv`

## Phase 4: Proof system

### Goal

Turn historical work into a repeatable selling asset without over-claiming package history.

### Status

Implemented structurally, expanded through the proof library in this pass.

### Deliverables

- `proof_inventory.md`
- `historical_client_package_mapping.csv`
- `selling_proof_library.md`

## Phase 5: Surface alignment

### Goal

Make the website, outreach, messaging, and packaging tell the same story.

### Status

Partially implemented in this pass.

### Deliverables

- aligned homepage and pricing logic
- aligned contact and about-page positioning
- aligned outreach guardrails
- `offer_surface_alignment.md`

## Phase 6: Marketing operating system

### Goal

Define how product and marketing run every week, every month, and every quarter.

### Status

Implemented as an operating framework in this pass.

### Deliverables

- `messaging_house.md`
- `outbound_message_matrix.md`
- `marketing_operating_system.md`

## Phase 7: Ongoing work after this cleanup

### Still needed

- enrich `clients_master.csv` with validated companies and buyers
- add more proof blocks as new case studies become public
- keep website copy aligned with the canon whenever pricing or packaging changes
- keep CRM fields aligned with the commercial system

### Success condition

The agency is operationally organized when every new prospect can be answered consistently across:

- package fit
- proof angle
- message angle
- website page
- proposal
- outreach angle
